# Architecture Decision Records (ADRs)

Each significant, hard-to-reverse decision is captured as a short, numbered Markdown
file so reviewers (and future us) can see _what_ was decided and _why_, including the
alternatives that were rejected.

- Files are named `NNNN-short-title.md` (zero-padded, incrementing).
- Status is one of `Proposed`, `Accepted`, `Superseded by NNNN`, `Deprecated`.
- Keep them short; link to code and other ADRs rather than duplicating detail.
