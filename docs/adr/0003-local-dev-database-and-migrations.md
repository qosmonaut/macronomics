# ADR 0003 — Local dev database and migrations

- **Status:** Accepted
- **Date:** 2026-06-03
- **Deciders:** Project owner (@qosmonaut)
- **Reviewers:** Project owner, OpenAI Codex
- **Relates to:** [ADR-0001](0001-initial-technology-stack.md) (Supabase EU + Drizzle)

## Context

M1 needs to build and **verify** the ingestion pipeline (schema → upsert → sort query) before
a Supabase project exists, without forcing Docker or an account on the developer.

## Decision

- **One Drizzle schema (`packages/db`), two drivers behind `createDb()`:**
  - With `DATABASE_URL` → **postgres-js** against Supabase EU via the Supavisor pooler
    (`prepare: false`, per ADR-0001 §6). This is production.
  - Without `DATABASE_URL` → **pglite** (embedded Postgres, in-memory or `PGLITE_DIR`) for
    local dev and verification. pglite and its Drizzle adapter are **devDependencies**,
    imported lazily so production never loads them.
- **Migrations** are generated by `drizzle-kit generate` and applied by the Drizzle migrator
  (same SQL files for both drivers). Committed under `packages/db/migrations/`.
- **Build-script approval:** pnpm 11 blocks dependency build scripts by default; `esbuild`
  (pulled by `drizzle-kit`) is allowed via `allowBuilds: { esbuild: true }` in
  `pnpm-workspace.yaml` (the package.json `pnpm` field is no longer read by pnpm 11).
- The app type standardizes on `PostgresJsDatabase`; the pglite handle is cast to it (same
  Postgres dialect), so app/query code is driver-agnostic.

## Consequences

- ✅ The whole pipeline is verifiable locally with **zero external infra** (`node services/ingestion/src/seed.ts`).
- ✅ Switching to Supabase EU is a `DATABASE_URL` swap — no code change.
- ⚠️ pglite is Postgres-compatible but **not identical** (no extensions, single-connection);
  treat it as a dev/test convenience, and run a real ingestion against Supabase before relying
  on prod behavior. Schemas here are plain (no exotic types), so divergence risk is low.
- ⚠️ `prepare: false` is required for the pooler; keep it when adding queries.
